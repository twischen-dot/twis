## 二、逐章写作要点与关键问题

### 1. 投资结论（Executive Summary）

**最后写，放在开头。**  
目标：让读者在 1–2 页内清楚知道：

- 你对 GOOG 的 **评级**（看多/中性/看空）
- 你对未来 3–5 年的 **核心判断**
- 最重要的 **驱动因素与风险**

可包含的结构：

1. **一句话结论**
    
    - 例：
        
        > 在我们基准情景下，Google 未来 3–5 年归母净利 CAGR 约为 x%，当前估值对应 P/Eforward​≈y，处于（合理/低估/高估）区间，我们给予（增持/持有）意见。
        
2. **核心论点（3–5 条 bullet）**
    
    - 搜索 & YouTube 的护城河是否仍牢固？
    - Cloud 与 AI 能否提供新的增长极？
    - 监管与 AI 竞赛会否侵蚀利润率？
3. **简要估值结论**
    
    - 基准情景目标价范围：  
        Target Price (12–24m)=X∼Y USD
    - 对应的上涨/下跌空间。
4. **关键风险 & 触发点**
    
    - AI 替代传统搜索的速度超预期；
    - 监管处罚/行为纠正导致搜索默认位置改变；
    - Cloud 盈利改善不及预期等。

### 2. 公司概览与商业模式

目标：让读者快速理解 **Google 的“钱从哪来、为什么能赚”**。

建议包含：

1. **公司简介与历史节点**
    
    - 重要里程碑：搜索 → Android → YouTube → Cloud → AI 转型（Gemini）。
    - 公司结构：Alphabet 旗下 Google、Other Bets（Waymo 等）。
2. **收入结构与商业模式**
    
    - 用最近一年/最近四个季度的数据做一张饼图/柱状图：
        - 按业务：Search & Other、YouTube Ads、Google Network、Google Cloud、Other。
    - 简要说明每块业务如何变现：广告竞价机制、点击计费、订阅、云服务计费等。
3. **护城河概览（只做提纲，详细展开可放后文）**
    
    - 网络效应（搜索 + 广告主 + 用户）
    - 数据和算法优势
    - 生态锁定（Android、Chrome、Gmail、YouTube 等入口）

### 4. 行业与竞争格局

核心问题：**Google 在什么战场上，与谁竞争，战局如何演变？**

1. **行业结构与价值链**
    
    - 在线广告行业：搜索广告、展示广告、视频广告、社交广告。
    - 云计算行业：IaaS / PaaS / SaaS 格局。
2. **主要竞品对比**
    
    - 搜索 & 广告：微软（Bing + OpenAI）、Meta（广告算法和用于电商）、亚马逊（站内搜索广告）。
    - Cloud：AWS、Azure 与 Google Cloud 的市场份额、增速、利润率对比。
    - AI：OpenAI、Anthropic、Meta（Llama）、其他开源力量。
3. **结构性变化趋势**
    
    - 用户行为从传统搜索向：
        - 社交平台搜索（Instagram、TikTok 等）
        - 电商平台搜索（Amazon）
        - AI 助手/Chatbot 的迁移。
    - 广告预算从线下向线上、从传统搜索到视频/短视频/社交的迁移。
4. **Google 的相对优势与劣势**
    
    - 优势：入口、数据、算力、自研芯片、现金流。
    - 劣势：组织结构庞大、出手相对谨慎、部分 AI 产品落地节奏被市场质疑等。

建议用一张对比表格，列出 MSFT / META / AMZN / GOOG 在广告、Cloud、AI 模型、开发者生态上的对比（打勾或星级）。

2. **模型与技术栈**

    - 与 GPT 系列、Claude、Llama 等的能力对比（可以引用基准测试、但不必太细）。

3. **AI 产品线与商业化路径**

### 6. 财务分析与估值

这是长期基本面投资的“核心工地”。

1. **历史财务概览**
    
    - 过去 5 年：收入、营业利润、净利润、EPS、自由现金流（FCF）的趋势图。
    - 盈利能力：
        - 毛利率、营业利润率、净利率
        - FCF Margin：FCF Margin=营业收入自由现金流​
    - 资本效率：
        - ROIC 或 ROE 的变化趋势
        - 股权激励（SBC）占收入/利润比例。
2. **分部财务拆解**
    
    - 如有披露，拆分 Google Services、Google Cloud、Other Bets 的收入和营业利润。
    - 看看 Cloud 何时从亏损转为盈利，利润率提升斜率如何。
3. **简化预测模型（3–5 年）**  
    你不必做极度精细的模型，可以用“自上而下 + 关键假设”的方式：
    
    - 收入预测：  
        Revenuet+1​=∑i​Revenuei,t​×(1+gi​)  
        其中 i 为各业务分部，gi​ 为你假设的中期 CAGR 或逐年增速。
    - 利润预测：
        - 假设每块业务的目标营业利润率，或者整体利润率随 AI 投入/效率提升的变化趋势。
    - FCF 预测：  
        FCF≈Net Income+D&A−CAPEX−ΔWorking Capital  
        可以用“FCF / Revenue 比例”简化。
4. **估值方法选择**
    
    至少做两种角度，互相校验：
    
    - ① **相对估值**
        
        - 对比同业（MSFT、META、AMZN）的：  
            P/Eforward​, EV/EBITDA, EV/FCF
        - 看看在你预期的 EPS 增速下，当前估值是偏贵还是偏便宜。
    - ② **简化 DCF 或 SOTP**
        
        - 若做 DCF：
            - 预测 5 年 FCF，给出一个长期增长率 g，折现率用 WACC。
            - 不要求极精准，更重要是“对假设敏感性”的理解。
        - 若做 SOTP：
            - 给 Search & YouTube 一个成熟业务估值倍数；
            - 给 Cloud 一个高成长云计算估值倍数；
            - Other Bets 按期权价值或协调估值处理。
5. **估值敏感性与安全边际**
    
    - 改变关键假设（如 Cloud 增速从 25% 换成 18%；AI CAPEX 效率不及预期），看估值变化。
    - 对比当前股价，看看安全边际是否足够（例如你希望有至少 20–30% 的低估空间才出手）。

### 7. 监管、法律与关键风险

这里要把“坏消息”系统性列出，并说明：哪些你认为是“价格已反映”，哪些可能尚未充分反映。

1. **主要监管与诉讼**
    
    - 美国与欧盟的反垄断案件（搜索、广告技术堆栈）。
    - 潜在的处罚方式：罚款 vs. 改变业务行为（默认搜索位置、广告竞价机制等）。
2. **隐私 & 数据保护**
    
    - GDPR、CCPA 等对用户数据收集和广告定向的影响。
    - Cookie 政策变化、浏览器隐私功能对广告业务的冲击。
3. **AI 特有风险**
    
    - 内容审核、错误信息、模型偏见引发的声誉与合规风险。
    - 训练数据版权与知识产权诉讼。
4. **其他关键风险**
    
    - 管理层变动、内部文化与执行力问题。
    - 巨额回购 + 大规模 AI CAPEX 可能造成财务灵活性下降。

每条风险后面可以简单标明：

- 发生概率：高/中/低
- 影响程度：高/中/低  
    并指出：你是否认为市场已充分定价该风险。

### 8. 情景分析（Bear / Base / Bull）

把前文的核心假设和不确定性“打包”成 3 个情景，让你的结论更透明。

1. **基准情景（Base Case）**
    
    - 你认为最有可能发生的情况：
        - 每块业务的增速区间
        - Cloud 利润率改善速度
        - AI 投入与回报大致匹配
2. **乐观情景（Bull Case）**
    
    - AI 在 Search 和 Cloud 端的变现快于预期；
    - 监管风险相对温和，不需要结构性改变商业模式。
3. **悲观情景（Bear Case）**
    
    - AI/新搜索范式显著侵蚀传统搜索广告份额；
    - Cloud 竞争加剧导致盈利能力难以提升；
    - 监管强制行为改变（默认搜索剥离等）。

为每个情景给出：

- 关键财务指标（例如第 5 年 EPS / FCF）
- 对应估值与隐含股价区间
- 你主观赋予的概率（可选）

## 三、一个可执行的工作流程

给你一个从“现在”到“写完初稿”的实际步骤，你可以按阶段推进。

### 阶段 1：资料收集与整理（1–2 周）

- 从官方信源 & SEC（10-K、10-Q）下载最近 3–5 年的：
    - 年度/季度报表
    - 分部收入、利润数据
    - CAPEX、SBC、FCF、回购数据
- 收集 3–5 篇高质量卖方报告（不同机构，便于比较假设）。
- 从 Seeking Alpha / Substack 精选少量长文，重点看 AI、监管、竞争格局视角。

### 阶段 2：搭建数据表与图表（1 周）

- 在 Excel / Sheets 里整理：
    - 5 年财务指标表
    - 分部收入 & 利润表
    - 基础估值对比表（GOOG vs. MSFT/META/AMZN）
- 绘制核心图：收入与利润增长、分部占比变化、估值倍数历史区间。

### 阶段 3：写定性章节初稿（1–2 周）

- 先写：公司概览、业务分部拆解、行业竞争、AI 战略、监管风险。
- 每一章按照“问题清单”写，不求一遍完美，优先保证框架完整。

### 阶段 4：搭建预测与估值模型（1 周）

- 确定关键假设：
    - 各分部中期增速范围
    - 整体利润率、CAPEX/FCF 比例
- 做一个简化的 3–5 年预测表和 DCF / SOTP 模型。
- 对关键参数做敏感性测试，形成情景分析数据。

### 阶段 5：撰写投资结论与情景分析（1 周）

- 根据模型输出 + 定性分析，写：
    - 投资结论
    - 情景分析
    - 关键风险与触发点

### 阶段 6：整体审阅与简化

- 全文检查逻辑是否连贯：
    - 你在“结论”中引用的观点，是否都能在正文中找到依据？
    - 模型假设是否与行业、公司描述一致？
